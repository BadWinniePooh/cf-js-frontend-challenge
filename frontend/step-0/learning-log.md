<a id="step-0-top-takeaways"></a>

### Step 0 — Top takeaways

There are **9** learning goals this journey is aiming to give you. Read through them, and select the **two** most
relevant for you right now. Mark those with `[x]` in the list below.

- [ ] Build UIs from semantic structure first, then behavior (clear separation of concerns).
- [ ] Use Atomic Design thinking to decide component boundaries and responsibilities.
- [ ] Build and reason about Web Components lifecycle (connectedCallback, observed attributes, rerendering).
- [ ] Compose components with clear data flow (structured inputs, parent-to-child rendering).
- [ ] Implement event-driven pub/sub with bubbling events and orchestrator coordination (loose coupling).
- [ ] Move data concerns out of UI into storage/API layers (IndexedDB, loaders, backend fetch).
- [ ] Build forms with native browser capabilities (FormData, validation, required fields) before custom logic.
- [ ] Create advanced reusable controls with form-associated custom elements (ElementInternals).
- [ ] Treat architecture as replaceable boundaries: mock vs real backend (MSW), pull vs push updates (fetch/WebSocket) 
      without rewriting core components.
